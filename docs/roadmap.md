## 候选优化（按价值排）

### 准确性（识别质量，多人后更值得）

| 项 | 状态 | 说明 |
|---|---|---|
| VAD（M3.2） | ⭐❌ | 静音/非人声段不参与判定，避免拿静音 embedding 误判 → 减少误掐/漏掐 |
| 阈值迟滞 + 去抖（M3.3） | ⭐❌ | 开>0.5 / 关<0.4 + 短保持，压住多声纹边界相似度来回跳导致的掐声闪烁 |
| 每目标独立阈值 | ❌ | N 个声纹时全局 0.5 不一定都合适 |

### 音质

| 项 | 状态 | 说明 |
|---|---|---|
| 掐声淡入淡出 | ⭐❌ | 硬切静音可能"咔哒"；mute/unmute 加 5~10ms 斜坡，听感干净很多。小改动大收益 |
| 路由后老流不跟走 | ❌ | 持久路由对已在播的流不生效（有时要暂停重播）→ 检测并提示用户重启音频流 |

### 健壮性

| 项 | 状态 | 说明 |
|---|---|---|
| `stop()` 阻塞 UI ~200ms | ❌ | 线程 200ms 超时才查退出标志；改"停止时立即唤醒线程"或后台 join，点停止更跟手 |
| 抓取回调里加锁 | ❌ | 实时回调理论上不该加锁（极低概率爆音）；历史环改 try_lock / 无锁 |
| 运行中设备变更监听 | ❌ | 用户中途改设备会与还原冲突（`IMMNotificationClient`，价值不对等，记着） |

### 分发 / 上手

| 项 | 状态 | 说明 |
|---|---|---|
| 静态链接 MinGW 运行时 | ❌ | `-static` → release 包少带 3 个 DLL，更干净 |
| 代码签名 | ❌ | 现在 exe 没签名，用户撞 SmartScreen 警告 |
| 一键安装器（Inno/NSIS） | ❌ | 顺带装 VB-CABLE + 查 WebView2，大幅降上手门槛 |
| 打包脚本 + tag 自动发 release | ❌ | `package-release.ps1` + CI on tag，一键发版 |

### 代码质量 / CI

| 项 | 状态 | 说明 |
|---|---|---|
| gui_main JSON 字符串拼接 | ❌ | 稍脆，可换小 JSON helper |
| 补测试 | ❌ | AppRouter 打包/journal、多目标判定逻辑、VAD（做了之后） |
| CI actions Node20→24 | ❌ | 消除 GitHub 弃用警告（2026-06-16 前升级 checkout/cache 版本） |

## 建议顺序

1. **掐声淡入淡出** —— 小而立竿见影的听感提升
2. **VAD + 阈值迟滞** —— 剩下最大的识别质量杠杆，多人场景尤其明显
3. **分发打磨**（静态链接/签名/安装器）—— 准备更广泛发布时再做
