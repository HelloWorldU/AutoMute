# M2 执行计划：声纹识别 → 自动掐声

> 目标：把 M1 的"按 m 手动静音"换成"程序自动认出目标说话人、自动掐"。
> 状态见 [`../STATUS.md`](../STATUS.md)。本文档是 M2 的拆解与决策记录。

## 端到端目标

```
事先：给一段目标的声音 → 模型算出"声纹"（一个向量）→ 存起来
运行：抓取端分流音频 → 分析线程攒一小段 → 模型算当前说话人声纹
      → 和目标声纹做余弦相似度 → 超阈值=就是他 → 掐掉这段
```

## 子步骤（每步都要能单独跑、单独验证）

| # | 目标 | 验证标准 |
|---|------|----------|
| M2.1 | 集成 ONNX Runtime + dummy 推理 | 程序能链接 ORT、加载一个模型、跑一次前向不崩 |
| M2.2 | 真实声纹模型跑 wav | 喂一段 wav，打印出 embedding 向量（长度对、数值稳定） |
| M2.3 | 录入目标声纹 | 加载目标 wav → 算 embedding → 存文件；再加载回来一致 |
| M2.4 | 实时分析线程 | 抓取端分流到第二个 ring buffer；分析线程攒窗口→推理→余弦比对→打印"是他/不是他" |
| M2.5 | 判定驱动 gate | 判定结果替换手动 m 键，命中目标自动静音 |

## 关键决策（推荐值，待确认）

| 决策 | 推荐 | 理由 |
|------|------|------|
| 声纹模型 | **ECAPA-TDNN（ONNX 导出版）** | 准、文档多、有现成 ONNX；许可证多为 Apache/MIT，可商用。瘦身留到 M3 |
| ONNX Runtime 接入 | **官方预编译 Windows 二进制**（DLL+头） | 最省事，CMake 指向即可；放 `third_party/onnxruntime/`，写 `cmake/FindOnnxRuntime.cmake` 或直接 `target_link` |
| 推理跑在哪 | **独立分析线程 + 第二个 ring buffer** | 铁律：绝不在抓取/渲染回调里推理（会阻塞→爆音）。正好复用 M1 的 `SpscRingBuffer` |
| 录入输入 | **先支持加载 wav 文件** | 确定、可测、可复现；实时录制后补 |
| 判断窗口长度 | 先用 ~1s（M3 再压短） | 先求准、能跑通，延迟优化是 M3 的事 |

## 风险与注意

- **分析线程不能拖累音频线程**：抓取回调只做"把音频拷进第二个 ring buffer"，推理在分析线程异步做。
- **采样率/格式对齐**：声纹模型通常要 16kHz 单声道；而 WASAPI 给的是 48kHz 立体声 → 分析线程需**重采样 + 转单声道**（M2.4 的活）。
- **模型许可证**：选模型时确认是否可商用（关系到项目能否商用），ECAPA 系多为宽松许可。
